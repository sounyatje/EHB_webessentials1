    Structuur van een HTML-document

    Elke HTML-tag begint met <naam> en sluit met </naam>. Voorbeeld: <p>Tekst</p>

    Belangrijke hoofdelementen

    • <!DOCTYPE html> → Definieert dat het document een HTML5-bestand is (geen sluit-tag nodig)
    • <html> ... </html> → Hoofdelement van de pagina
    • <head> ... </head> → Bevat metadata en instellingen (titel, CSS, scripts)
    • <meta charset="UTF-8"> → Tekencodering instellen (geen sluit-tag nodig)
    • <body> ... </body> → Hoofdinhoud / de zichtbare tekst en elementen

    Orphelijn-tags (zonder sluit-tag)

    Er zijn ook zogenaamde orphelijn-tags die geen sluit-tag hebben. Voorbeelden:

    • <img src="afbeelding.jpg" alt="Beschrijving"> → Afbeelding
    • <br> → Nieuwe regel (line break)
    • <hr> → Horizontale lijn
    • <input type="text"> → Invoerveld (formulier)
    • <link rel="stylesheet" href="stijl.css"> → Externe CSS
    • <meta charset="UTF-8"> → Metadata
    • <area>, <base>, <col>, <command>, <embed>, <source>, <track>, <wbr> → Andere voorbeelden

Hoe ziet een webpagina eruit?

Alle zichtbare elementen van een webpagina bevinden zich in het <body> gedeelte.

  • <header> → Bovenste gedeelte van de pagina (logo, titel, menu)
  • <nav> → Navigatiebalk met links
  • <section> → Hoofdstuk of deel van de inhoud
  • <article> → Zelfstandig stuk inhoud (bv. blogartikel)
  • <aside> → Zijbalk met extra info (bv. reclame, links)
  • <footer> → Onderaan de pagina (contact, copyright)

Structuur

  • <!DOCTYPE html> → Definieert dat het document HTML5 is (geen sluit-tag)
  • <html> ... </html> → Hoofdelement van de pagina
  • <head> ... </head> → Metadata en instellingen
  • <meta charset="UTF-8"> → Tekencodering instellen (geen sluit-tag)
  • <title> ... </title> → Titel in de browser-tab
  • <body> ... </body> → Zichtbare inhoud van de pagina

Layout

  • <header> ... </header> → Bovenste gedeelte (logo, menu, titel)
  • <nav> ... </nav> → Navigatiemenu
  • <section> ... </section> → Deel van de inhoud
  • <article> ... </article> → Zelfstandig stuk inhoud
  • <aside> ... </aside> → Zij-inhoud (bv. reclame)
  • <footer> ... </footer> → Voettekst van de pagina
  • <div> ... </div> → Container/blok voor structuur
  • <span> ... </span> → Inline container voor tekst

Tekst

  • <h1> ... </h1> t/m <h6> ... </h6> → Titels (van groot naar klein)
  • <p> ... </p> → Paragraaf
  • <br> → Regelafbreking (geen sluit-tag)
  • <hr> → Horizontale lijn (geen sluit-tag)
  • <strong> ... </strong> → Belangrijke tekst (vet)
  • <em> ... </em> → Nadruk (cursief)
  • <sup> ... </sup> → Superscript (bv. 2<sup>3</sup> = 2³)
  • <sub> ... </sub> → Subscript (bv. H<sub>2</sub>O = H₂O)
  • <blockquote> ... </blockquote> → Lange citaat (meestal met inspringing)
  • <abbr title="..."> ... </abbr> → Afkorting met uitleg (tooltip bij hover)
  • <cite> ... </cite> → Naam van een bron of auteur
  • <dfn> ... </dfn> → Definities van een term (eerste keer uitleggen)
  • <ins> ... </ins> → Toegevoegde tekst (meestal onderstreept)
  • <del> ... </del> → Verwijderde tekst (meestal doorstreept)

Opmerking: Tags zoals <strong>, <em>, <b> en <i> zijn niet bedoeld om de tekst stijl te geven, maar hebben een semantische betekenis (inhoudelijk belang of nadruk). Voor echte vormgeving (vet, cursief, kleuren, lettertypes) gebruikt men **CSS**.

Links & media

  • <a href="..."> ... </a> → Hyperlink
  • <img src="..." alt="..."> → Afbeelding (geen sluit-tag)
  • <video src="..." controls> ... </video> → Video
  • <audio src="..." controls> ... </audio> → Audio
  • <a href="#section1">Ga naar sectie 1</a> → Ancre (scroll naar een element op de pagina)
  • <a href="tel:+32123456789">Bel ons</a> → Link naar telefoonnummer waarmee je direct kunt bellen
  • <a href="mailto:email@example.com">Stuur een e-mail</a> → Link naar je mailbox met de ontvanger al ingevuld

Uitleg over attributen en hun waarden

Een attribuut geeft extra informatie aan een HTML-element en staat binnen de starttag. Het beïnvloedt hoe het element zich gedraagt of wat het inhoudt.

De waarde van een attribuut vertelt het element welke specifieke informatie het moet gebruiken. Waarden staan altijd tussen aanhalingstekens " ".

Voorbeelden:

  • <img src="foto.jpg" alt="Beschrijving van de foto"> → src en alt zijn attributen, met respectievelijke waarden.
  • <a href="https://voorbeeld.com">Klik hier</a> → href is een attribuut met de waarde van de URL.
  • <input type="text" name="voornaam" placeholder="Vul je naam in"> → type, name en placeholder zijn attributen met bijbehorende waarden.

Belang van attributen: Ze maken je pagina functioneel, toegankelijk en correct voor SEO.

Lijsten

  • <ul> ... </ul> → Ongeordende lijst (bolletjes)
  • <ol> ... </ol> → Geordende lijst (nummers)
  • <li> ... </li> → Lijst-item
  • <dl> ... </dl> → Definitielijst
  • <dt> ... </dt> → Term in een definitielijst
  • <dd> ... </dd> → Beschrijving in een definitielijst

Tabellen

  • <table> ... </table> → Tabel
  • <tr> ... </tr> → Rij
  • <th> ... </th> → Tabelkop
  • <td> ... </td> → Tabelcel
  • <thead> ... </thead> → Kop van de tabel
  • <tbody> ... </tbody> → Lichaam van de tabel
  • <tfoot> ... </tfoot> → Voet van de tabel

Formulieren

  • <form> ... </form> → Formulier
  • <input type="text"> → Tekstinvoer (geen sluit-tag)
  • <input type="password"> → Wachtwoordinvoer
  • <input type="checkbox"> → Checkbox
  • <input type="radio"> → Radioknop
  • <input type="submit"> → Verstuurknop
  • <label> ... </label> → Label voor een invoerveld
  • <textarea> ... </textarea> → Meerdere regels tekstveld
  • <select> ... </select> → Keuzelijst
  • <option> ... </option> → Optie in een keuzelijst
  • <button> ... </button> → Knop

Semantisch extra

  • <main> ... </main> → Hoofdinhoud van de pagina
  • <figure> ... </figure> → Groep afbeelding + bijschrift
  • <figcaption> ... </figcaption> → Bijschrift bij een afbeelding
  • <mark> ... </mark> → Gemarkeerde tekst
  • <code> ... </code> → Codefragment
  • <pre> ... </pre> → Voorgeformatteerde tekst (zoals geschreven)

Opmerking: Veel ontwikkelaars hebben de slechte gewoonte om te veel <div> elementen in elkaar te zetten (“divs op divs op divs...”). Dit maakt de code moeilijk leesbaar en minder overzichtelijk.
